<threat_model>

## Trust Boundaries

| Boundary                | Description                                                                                         |
| ----------------------- | --------------------------------------------------------------------------------------------------- |
| User filesystem ↔ App   | User-selected download directories; fs scope restricts to $HOME/$APPDATA                            |
| Frontend ↔ Rust Backend | Tauri IPC invoke/event bridge; all commands typed, plugin permissions allow-listed                  |
| Config ↔ Runtime        | tauri.conf.json and capabilities/default.json define app constraints; modifications require rebuild |

## STRIDE Threat Register

| Threat ID | Category               | Component                      | Disposition | Mitigation Plan                                                                                              |
| --------- | ---------------------- | ------------------------------ | ----------- | ------------------------------------------------------------------------------------------------------------ |
| T-01-01   | Tampering              | tauri.conf.json / capabilities | mitigate    | capabilities/default.json uses allow-lists per plugin; default-deny for all commands not explicitly granted  |
| T-01-02   | Information Disclosure | tauri.conf.json                | accept      | No secrets in config file; all values are non-sensitive app configuration                                    |
| T-01-03   | Elevation              | fs plugin scope                | mitigate    | fs scope restricted to $APPDATA/** and $HOME/** via capabilities configuration; arbitrary path access denied |
| T-01-04   | Tampering              | package.json dependencies      | mitigate    | All versions exact-pinned (no ranges) preventing supply-chain drift; bun.lock provides integrity hashes      |

</threat_model>
