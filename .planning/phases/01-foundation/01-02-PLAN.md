**Step B: Create .lintstagedrc.mjs** — maps staged file patterns to lint/format commands:

```javascript
/** @type {import('lint-staged').Config} */
export default {
  '*.{ts,vue}': ['eslint --fix', 'prettier --write'],
  '*.{js,mjs,cjs}': ['eslint --fix', 'prettier --write'],
  '*.{json,md,css,html}': ['prettier --write'],
  '*.rs': ['cargo fmt --manifest-path src-tauri/Cargo.toml -- --check'],
};
```

Key decisions:

- TypeScript/Vue files get both ESLint fix and Prettier format
- JSON/MD/CSS/HTML get only Prettier (ESLint doesn't handle these)
- Rust files get cargo fmt --check (we only check, not auto-fix on commit — Rust formatting changes are too significant to auto-apply blindly)
- The `--manifest-path` ensures cargo fmt runs in the src-tauri workspace context

**Step C: Create .github/workflows/ci.yml** per D-10. The workflow runs on push to any branch and on PRs to master:

```yaml
name: CI

on:
  push:
    branches: ['**']
  pull_request:
    branches: ['master']

env:
  BUN_VERSION: '1.3.2'

jobs:
  frontend-checks:
    name: Frontend Checks
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: ${{ env.BUN_VERSION }}

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: TypeScript type check
        run: bun run typecheck

      - name: ESLint
        run: bun run lint

      - name: Prettier format check
        run: bun run format:check

      - name: Frontend tests
        run: bun run test

  backend-checks:
    name: Backend Checks
    runs-on: ubuntu-latest
    timeout-minutes: 15

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Rust
        uses: dtolnay/rust-toolchain@stable
        with:
          toolchain: stable
          components: rustfmt, clippy

      - name: Cache Rust dependencies
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            src-tauri/target
          key: ${{ runner.os }}-cargo-${{ hashFiles('src-tauri/Cargo.lock') }}
          restore-keys: |
            ${{ runner.os }}-cargo-

      - name: Install system dependencies (Tauri)
        run: |
          sudo apt-get update
          sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf libsoup-3.0-dev libjavascriptcoregtk-4.1-dev

      - name: Cargo fmt check
        run: cargo fmt --manifest-path src-tauri/Cargo.toml -- --check

      - name: Cargo clippy
        run: cargo clippy --manifest-path src-tauri/Cargo.toml -- -D warnings

      - name: Cargo check
        run: cargo check --manifest-path src-tauri/Cargo.toml

      - name: Cargo test
        run: cargo test --manifest-path src-tauri/Cargo.toml
```

Key decisions:

- Frontend and backend checks run as separate parallel jobs
- Bun version pinned to 1.3.2 (matches local dev environment from RESEARCH.md Environment Availability)
- `--frozen-lockfile` prevents CI from modifying lockfile
- `dtolnay/rust-toolchain@stable` is the community-preferred Rust setup action (faster than actions-rs)
- System deps for Tauri on Linux: libwebkit2gtk-4.1-dev, libappindicator3-dev, librsvg2-dev, patchelf, libsoup-3.0-dev, libjavascriptcoregtk-4.1-dev (required for Tauri v2 on Ubuntu runners)
- Cargo cache step speeds up subsequent runs by caching ~/.cargo and target/

| ----------------------- | -------------------------------------------------------------------- |
| CI Runner ↔ Repository  | GitHub Actions accesses source code; workflow file defines what runs |
| Git hook ↔ Staged files | pre-commit hook runs lint-staged on files before they enter the repo |

## STRIDE Threat Register

| Threat ID | Category               | Component               | Disposition | Mitigation Plan                                                                                                                                              |
| --------- | ---------------------- | ----------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| T-02-01   | Tampering              | GitHub Actions workflow | mitigate    | CI workflow uses pinned action versions (checkout@v4, setup-bun@v2, cache@v4, dtolnay/rust-toolchain@stable); no third-party actions with unknown provenance |
| T-02-02   | Information Disclosure | CI logs                 | accept      | CI runs on public GitHub Actions runners; no secrets in Phase 1 (no API keys, tokens, or credentials needed)                                                 |
| T-02-03   | Denial of Service      | CI minutes              | accept      | Public repo on GitHub gets unlimited CI minutes; timeout limits set (10min frontend, 15min backend) prevent runaway jobs                                     |

</threat_model>
