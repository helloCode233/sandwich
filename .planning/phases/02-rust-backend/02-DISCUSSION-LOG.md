## 种子生成策略

| Option | Description | Selected |
|--------|-------------|----------|
| 纯随机，不可编辑 | 点击生成即随机种子，操作链透明但不可修改 | ✓ |
| 随机生成 + 可手动调整 | 随机生成后可手动调整参数 | |

## 状态持久化

| Option | Description | Selected |
|--------|-------------|----------|
| 全部持久化 | 种子+队列+元数据+输出目录偏好 | ✓ |
| 仅持久化种子 | 种子持久化，队列每次清空 | |
| 会话级别 | 不持久化，关闭即丢失 | |

**User's choice:** 全部持久化
**Notes:** 重启恢复关闭前状态。队列中文件路径检查有效性，已移动/删除的标记失效。

## 批处理模型

| Option | Description | Selected |
|--------|-------------|----------|
| 顺序处理 | 逐个排队，CPU/内存可控 | |
| 并行处理（2-3并发） | 同时处理多个，速度快但内存高 | |

**User's choice:** 用户可选并发数
**Notes:** 最灵活但增加 Phase 3 UI 复杂度（需添加并发设置项）。

## 视频导入限制

| Option | Description | Selected |
|--------|-------------|----------|
| FFmpeg 通用格式 | mp4, mov, avi, mkv, webm, flv, wmv | ✓ |
| 仅 mp4+mov | 最严格 | |
| 无大小上限 | 不限制 | ✓ |
| 2GB 上限 | 防止误导超大文件 | |
| 导入时校验 | ffprobe 验证有效性 | ✓ |
| 处理时才发现 | 导入不校验 | |
| 允许重复 | 同路径可重复添加 | ✓ |
| 自动去重 | 同路径拒绝重复 | |

**User's choice:** 通用格式 + 无上限 + 导入校验 + 允许重复
**Notes:** 用户可能用不同种子处理同一源文件多次。
