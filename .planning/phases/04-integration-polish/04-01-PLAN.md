<objective>
Fix the event naming collision between executor.rs and batch.rs (both emit under "batch-progress" with different payload shapes) and enrich the Rust executor to emit rich per-file progress data (frame, FPS, ETA) via a new "batch-file-progress" event carrying a dedicated PerFileProgress struct.

Purpose: The current executor emits `ExecutorProgress { file, stage, percent }` under "batch-progress", which collides with batch.rs's `BatchProgress { total, completed, succeeded, failed, current_file }` under the same event name. The executor currently only extracts `time` from ffmpeg-sidecar's `FfmpegProgress` to compute a crude percentage -- frame, FPS, and speed are available but unused. This plan fixes both issues by introducing the `batch-file-progress` event with full progress data.

Output: Modified executor.rs that emits rich `PerFileProgress` under "batch-file-progress"; new `PerFileProgress` struct in models/batch.rs.
</objective>

```rust
/// Per-file frame-level progress emitted during FFmpeg execution.
/// Emitted via "batch-file-progress" event from executor.rs.
#[derive(Debug, Clone, Serialize)]
#[serde(rename_all = "camelCase")]
pub struct PerFileProgress {
    /// Display filename being processed.
    pub file: String,
    /// Percentage complete for this file (0.0 - 100.0).
    pub percent: f64,
    /// Current frame number being encoded (from ffmpeg-sidecar FfmpegProgress.frame).
    pub current_frame: u32,
    /// Total frames in this file (computed from VideoEntry.metadata.duration_secs * fps).
    pub total_frames: u32,
    /// Frames per second encoding speed (from ffmpeg-sidecar FfmpegProgress.fps).
    pub fps: f32,
    /// Estimated remaining seconds for this file (computed from (total_duration - current_time) / speed).
    pub remaining_seconds: f64,
}
```

Note: `PerFileProgress` only needs `Serialize` (not `Deserialize`) because it is emitted from Rust to frontend, never received. The existing `BatchProgress`, `BatchResult`, and `FileResult` have `Deserialize` because they are also received via Tauri commands -- `PerFileProgress` is event-only.
