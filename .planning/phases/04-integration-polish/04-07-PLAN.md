<objective>
Fix the `labelText` computed in BatchBanner.vue so it correctly distinguishes between batch completion and batch cancellation. Currently it uses `batchStore.lastResult?.failed.length` as a proxy for cancellation, which causes a batch that fully completes with some failed files (e.g., 3 succeeded + 2 failed) to display "Batch Cancelled" instead of "Batch Complete."

The BatchSummary component already has correct cancellation detection logic using `(succeeded.length + failed.length) < progress.total`. This plan applies the same formula to BatchBanner.

Purpose: After processing finishes, the banner correctly labels the outcome. The ROADMAP SC-2 requirement ("completion summary accurately reflects batch outcome") is partially broken by this UI mislabel.
Output: BatchBanner with a new `wasCancelled` computed and corrected `labelText`.
</objective>
