## Spacing Scale

Declared values (multiples of 4, matching UnoCSS presetUno defaults):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Resize handle width, inline icon gaps |
| sm | 8px | Compact element spacing (gap-2) |
| md | 12px | BatchControls internal spacing (py-3, space-y-3) |
| lg | 16px | Default section padding (px-4), default element gaps |
| xl | 24px | Section separation, modal padding |
| 2xl | 32px | Major layout gaps |
| 3xl | 48px | Empty state icon size, large section breaks |
| 4xl | 64px | Page-level spacing |

Note: UnoCSS `presetUno()` defaults to a 4px base unit (`1 = 4px`). The values above are the physical pixel equivalents for the utility classes used in this phase.

Exceptions:
- Icon-only touch targets for cancel button use Naive UI's `size="large"` (34px height)
- BatchBanner inner padding: 10px (not on 4px grid — inherited from Phase 3, acceptable as banner is compact)

## Typography

| Role | Size | Weight | Line Height | Naive UI Token |
|------|------|--------|-------------|-----------------|
| Metadata label | 12px | 400 | 1.5 | NText `depth="3"` + class `text-xs` |
| Body / UI text | 14px | 400 | 1.5 | NText default + class `text-sm` |
| Section heading | 14px | 600 | 1.4 | NText `strong` + class `text-sm` |
| Summary heading | 16px | 600 | 1.3 | NText `strong` + class `text-base` |
| Progress counter | 12px | 400 | 1.0 | NText `depth="2"` + class `text-xs tabular-nums` |

Weights: exactly 2 — regular (400) and semibold (600).

Summary heading uses 16px because the completion modal is a distinct information-dense view that benefits from one step larger type.

## Color

Naive UI dark theme provides the foundation. All colors below exist within the dark theme context.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#101014` / `var(--n-color)` | Page background, panel surfaces, modal backdrop |
| Secondary (30%) | `#1a1a1f` | Queue item cards, completion summary card rows |
| Banner surface | `#1a1a24` | BatchBanner background (processing state indicator) |
| Accent (10%) | `#2080f0` | Progress bar active color, resize handle hover, selected state, primary button |
| Success | `#18a058` | 100% progress bar, succeeded file count, checkmark icons |
| Destructive | Naive UI `type="error"` (≈ `#d03050`) | Cancel button, failed file indicators, error text |
| Warning | Naive UI `type="warning"` (≈ `#f0a020`) | Invalid file tags, partial progress states |
| Border | `#2a2a36` / `var(--n-border-color)` | Panel dividers, card edges, banner separator |

Accent reserved for:
- Overall progress bar fill (in-progress state, < 100%)
- Resize handle hover/active state
- Seed/queue empty state icon
- Selected file indicator during processing (current file highlight border)
- NOT used for: success indicators, text links, general highlights

Success green (`#18a058`) reserved for:
- Overall progress bar at 100%
- "Succeeded" count in completion summary
- CheckCircle icons next to succeeded files

Destructive reserved for:
- Cancel button (NButton `type="error"`)
- Failed file count and error text in completion summary
- AlertCircle / XCircle icons next to failed files
- Clear all button (queue)

### BatchSummary Component Contract

**Trigger:** `batchStore.isComplete === true && batchStore.lastResult !== null`

**Layout:**
- Renders inside the right panel, replacing BatchBanner's position (top of queue-area)
- Not a modal overlay — it is an inline summary panel that pushes queue content down
- Height: auto, expands to show full file list
- Collapsible: user can dismiss it, returning to BatchBanner (which shows "Complete" state at 100%)

**Structure (top to bottom):**
1. Status icon (CheckCircle green or AlertCircle warning) + title text ("Batch Complete" or "Batch Cancelled")
2. Summary line (e.g., "3 succeeded, 1 failed out of 4 files")
3. Succeeded section: heading + scrollable list of `{filename} -> {outputPath}` rows
4. Failed section (only if `failed.length > 0`): heading + scrollable list of `{filename}: {errorMessage}` rows
5. "Done" button (NButton, right-aligned) — calls `batchStore.resetBatch()` to return to idle state

**States:**
- Success only (all files succeeded): green checkmark, no failed section, green progress bar at 100%
- Partial success (some failed): warning icon, both sections shown
- Cancelled (incomplete batch): AlertCircle icon, shows what completed before cancellation
- All failed (0 succeeded): error icon, only failed section shown

**Scrolling:** Succeeded and failed lists scroll independently within the summary panel. Max height per list: 200px, then NScrollbar.

### Per-File Progress Bar Contract (in QueueList)

**Trigger:** `batchStore.isProcessing === true`

**Per queue item:**
- When `batchStore.progress.currentFile === entry.filename`: show NProgress bar below metadata line
- NProgress: `type="line"`, `indicator-placement="inside"`, `:height="18"`, `:color="#2080f0"`
- Progress percentage from per-file event (see Data Contract below)
- Below progress bar: frame counter text (left) + ETA text (right), both `text-xs` `depth="3"`

**Non-current queue items:**
- Show a subtle inactive state (reduced opacity or no progress bar)
- Overall completed items show a green checkmark overlay on the file card

**Interaction lock:**
- During processing, remove button per item is disabled (NButton `:disabled="true"`)
- ImportZone is disabled/hidden during processing
- Add Video button is disabled

## Interaction States

### Idle State
- BatchBanner: hidden (`v-if="false"`)
- QueueList: normal interactive mode (remove, add, clear enabled)
- BatchControls: Start button visible and enabled (if seed selected + queue not empty)

### Processing State
- BatchBanner: visible, shows "Processing {completed}/{total}", NProgress bar active
- BatchControls: Cancel button visible (replaces Start), all other controls disabled
- QueueList: per-file progress bars active, remove buttons disabled, ImportZone hidden
- Store: `isProcessing === true`, `progress` streamed via events

### Cancelling State (transitional, between cancel click and batch-cancelled event)
- BatchBanner: NProgress shows current progress, label changes to "Cancelling..."
- BatchControls: Cancel button disabled (shows loading/processing state)
- QueueList: current file progress freezes, remaining files show "Pending" or fade
- This state lasts until `batch-cancelled` event arrives (typically < 2 seconds)

### Complete State
- BatchBanner: NProgress at 100% (green), shows "Batch Complete" or "Batch Cancelled"
- BatchSummary: rendered below/alongside banner with full file breakdown
- BatchControls: Start button returns, all controls re-enabled
- QueueList: interactive mode restored, per-file progress bars removed
- Store: `isProcessing === false`, `isComplete === true`, `lastResult` populated

### Error During Processing (file-level, not batch-level)
- Toast notification appears for each file error (via `batch-file-error` event)
- Failed file marked in queue with error indicator (red border or NTag)
- Processing continues to next file (failure isolation)
- Failed files appear in completion summary under "Failed" section

## Data Contract

### Existing Types (no change)

```typescript

export interface BatchProgress {
  total: number;
  completed: number;
  succeeded: number;
  failed: number;
  currentFile: string | null;
}

export interface BatchResult {
  succeeded: string[];
  failed: FileResult[];
}

export interface FileResult {
  file: string;
  seed: string;
  error: string;
}
```

### New Type: PerFileProgress

Phase 4 requires per-file frame-level progress for the success criteria "percentage complete, current frame, and estimated remaining time." This data must arrive via a NEW event (`batch-file-progress`) emitted from the Rust backend during FFmpeg execution.

```typescript
// src/types/batch.ts — ADD to existing file
export interface PerFileProgress {
  /** Display filename being processed. */
  file: string;
  /** Percentage complete for this file (0-100). */
  percent: number;
  /** Current frame number being encoded. */
  currentFrame: number;
  /** Total frames in this file (from ffprobe metadata). */
  totalFrames: number;
  /** Frames per second encoding speed. */
  fps: number;
  /** Estimated remaining seconds for this file. */
  remainingSeconds: number;
}
```

### New Event: batch-file-progress

**Rust emission:** `app.emit("batch-file-progress", PerFileProgress { ... })`
**Frontend listener:** `listen<PerFileProgress>('batch-file-progress', ...)` in `useBatch` composable
**Store update:** New `perFileProgress` ref of type `Map<string, PerFileProgress>` in `useBatchStore`

### Store Extension (useBatchStore)

Add to existing store:
```typescript
/** Per-file progress map keyed by filename. */
const perFileProgress = ref<Map<string, PerFileProgress>>(new Map());

/** Current file's frame-level progress (null when idle). */
const currentFileProgress = computed(() => {
  if (!progress.value.currentFile) return null;
  return perFileProgress.value.get(progress.value.currentFile) ?? null;
});
```

### Composable Extension (useBatch)

Add event listener:
```typescript
let fileProgressUnlisten: UnlistenFn | null = null;

// In subscribe():
fileProgressUnlisten = await listen<PerFileProgress>('batch-file-progress', (event) => {
  store.setPerFileProgress(event.payload);
});
```

### Event Flow Summary

| Event | Payload | UI Update |
|-------|---------|-----------|
| `batch-progress` | `BatchProgress` | BatchBanner overall bar, completed counter |
| `batch-file-progress` | `PerFileProgress` | QueueList per-file bar, frame counter, ETA |
| `batch-file-error` | `FileResult` | Toast notification, failed file indicator |
| `batch-complete` | `BatchResult` | BatchSummary display, return to complete state |
| `batch-cancelled` | `BatchResult` | BatchSummary with cancellation notice |
