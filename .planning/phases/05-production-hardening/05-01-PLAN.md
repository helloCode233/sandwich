**Change 2: Add bundle key**
Insert a `"bundle"` key as a new top-level sibling after the `"plugins"` key (after line 32, before the closing `}`), with this exact content:

```json
  "bundle": {
    "active": true,
    "targets": "all",
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.png"
    ],
    "windows": {
      "nsis": {
        "installMode": "currentUser"
      }
    },
    "linux": {
      "deb": {
        "depends": []
      },
      "appimage": {
        "bundleMediaFramework": false
      }
    }
  }
```

Per D-01: `targets: "all"` with `windows` (NSIS for .exe + WiX for .msi) and `linux` (deb + AppImage) sub-configs. Per D-03: existing macOS .dmg build continues to work because Tauri v2 auto-generates .dmg when `targets: "all"` on macOS.
Per D-02: FFmpeg is NOT bundled in the installer (uses ffmpeg-sidecar auto_download at runtime). No `externalBin` or `resources` config needed.
Per RESEARCH Pitfall 1: Without explicit `bundle` key with `"active": true`, no installers are generated on non-macOS platforms.

The `icon` array references all four existing PNG icons. Tauri CLI auto-generates `.ico` (Windows) and `.icns` (macOS) during `cargo tauri build`.
