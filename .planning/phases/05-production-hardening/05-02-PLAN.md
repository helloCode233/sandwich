  <action>
Create NEW file `.github/workflows/build.yml` with this exact content:

```yaml
name: Build

on:
  push:
    branches: ['release']
  workflow_dispatch:

env:
  BUN_VERSION: '1.3.2'

jobs:
  build:
    permissions:
      contents: write
    strategy:
      fail-fast: false
      matrix:
        include:
          - platform: 'macos-latest'
            args: '--target aarch64-apple-darwin'
          - platform: 'macos-latest'
            args: '--target x86_64-apple-darwin'
          - platform: 'ubuntu-22.04'
            args: ''
          - platform: 'windows-latest'
            args: ''

    runs-on: ${{ matrix.platform }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install Ubuntu dependencies
        if: matrix.platform == 'ubuntu-22.04'
        run: |
          sudo apt-get update
          sudo apt-get install -y libwebkit2gtk-4.1-dev \
            libayatana-appindicator3-dev librsvg2-dev patchelf \
            libsoup-3.0-dev libjavascriptcoregtk-4.1-dev

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: ${{ env.BUN_VERSION }}

      - name: Setup Rust
        uses: dtolnay/rust-toolchain@stable
        with:
          toolchain: stable

      - name: Rust cache
        uses: swatinem/rust-cache@v2
        with:
          workspaces: './src-tauri -> target'

      - name: Install frontend dependencies
        run: bun install --frozen-lockfile

      - uses: tauri-apps/tauri-action@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          tagName: v__VERSION__
          releaseName: 'Sandwich v__VERSION__'
          releaseBody: 'See assets to download and install.'
          releaseDraft: true
          prerelease: false
          args: ${{ matrix.args }}
```

Per D-02: CI matrix uses `[macos-latest, ubuntu-22.04, windows-latest]`. FFmpeg is NOT bundled — `auto_download()` handles it at runtime. Per RESEARCH: use `ubuntu-22.04` (not ubuntu-latest/24.04) per RESEARCH Pitfall 2 and Tauri v2 official templates. `fail-fast: false` ensures one platform failure does not cancel others. `permissions: contents: write` required for tauri-action to create releases.

The `tauri-apps/tauri-action@v0` step:
- Reads version from `tauri.conf.json` (now "1.0.0" from Plan 05-01)
- Substitutes `__VERSION__` in `tagName` and `releaseName`
- Auto-creates a draft GitHub Release with all platform artifacts
- Each matrix job uploads its platform's bundles (.dmg / .deb + .AppImage / .msi + .exe)

macOS has two matrix entries: `aarch64-apple-darwin` (Apple Silicon) and `x86_64-apple-darwin` (Intel). Both run on `macos-latest` which is Apple Silicon with Rosetta 2 for x86_64 cross-compilation.

`bun install --frozen-lockfile` ensures reproducible builds (per existing ci.yml pattern).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner -> GitHub Release | GITHUB_TOKEN secret used by tauri-action to create releases |
| Build artifact -> User download | Installers downloaded from GitHub Releases (HTTPS, GitHub's infrastructure) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Information Disclosure | GITHUB_TOKEN in CI | mitigate | GitHub Actions automatically redacts secrets from logs. `GITHUB_TOKEN` is ephemeral (scoped to the workflow run). No long-lived tokens. |
| T-05-02-02 | Tampering | Build artifacts | accept | GitHub Releases provide checksums and are served over HTTPS. No code signing in v1.0 (deferred per CONTEXT.md). User must trust GitHub's infrastructure. |
| T-05-02-03 | Elevation of Privilege | CI workflow triggers | accept | Only `release` branch pushes and manual `workflow_dispatch` trigger builds. No PR-triggered builds (avoids untrusted code running in release pipeline). |
| T-05-02-04 | Denial of Service | CI minutes consumption | accept | Matrix builds consume ~40 minutes of CI time per run. `fail-fast: false` prevents wasted re-runs. Trigger limited to `release` branch pushes + manual dispatch. |
</threat_model>
