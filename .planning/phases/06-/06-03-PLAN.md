**Color Processing (4 functions):**

1. `build_hue_rotate_filter` — FFmpeg `hue` filter (RESEARCH Code Examples, lines 627-634)
   - Params: `hueAngle` (f64, degrees), `saturation` (f64, 0.5-1.5)
   - Clamp: hueAngle to [-90.0, 90.0], saturation to [0.5, 1.5]
   - Format: `hue=h={hueAngle}:s={saturation}`
   - Returns: `vec!["-vf".to_string(), filter]`

2. `build_saturation_adjust_filter` — FFmpeg `eq` filter
   - Params: `saturation` (f64, 0.5-2.0), `contrast` (f64, 0.8-1.3), `brightness` (f64, -0.2-0.2)
   - Clamp: saturation [0.5, 2.0], contrast [0.8, 1.3], brightness [-0.3, 0.3]
   - Format: `eq=saturation={saturation}:contrast={contrast}:brightness={brightness}`
   - Returns: `vec!["-vf".to_string(), filter]`

3. `build_brightness_contrast_filter` — FFmpeg `eq` filter (different params)
   - Params: `brightness` (f64, -0.3-0.3), `contrast` (f64, 0.7-1.5), `gamma` (f64, 0.8-1.3)
   - Clamp: brightness [-0.3, 0.3], contrast [0.7, 1.5], gamma [0.8, 1.3]
   - Format: `eq=brightness={brightness}:contrast={contrast}:gamma={gamma}`
   - Returns: `vec!["-vf".to_string(), filter]`

4. `build_color_balance_filter` — FFmpeg `colorbalance` filter (RESEARCH Code Examples, lines 636-642)
   - Params: `rs` (f64, red shadows), `gs` (f64, green shadows), `bs` (f64, blue shadows)
   - All three range [-0.3, 0.3], clamp to same
   - Format: `colorbalance=rs={rs}:gs={gs}:bs={bs}`
   - Returns: `vec!["-vf".to_string(), filter]`

**Noise Texture (3 functions):**

5. `build_film_grain_filter` — FFmpeg `noise` filter (RESEARCH Code Examples, lines 644-651)
   - Params: `strength` (u32, 0-100), `flags` (String, e.g., "t+u")
   - Clamp: strength to [5, 30]
   - Format: `noise=alls={strength}:allf={flags}`
   - Returns: `vec!["-vf".to_string(), filter]`

6. `build_gaussian_blur_filter` — FFmpeg `smartblur` or `gblur` filter
   - Params: `sigma` (f64, 0.5-5.0)
   - Clamp: sigma to [0.5, 3.0]
   - Format: `gblur=sigma={sigma}`
   - Returns: `vec!["-vf".to_string(), filter]`

7. `build_sharpen_filter` — FFmpeg `unsharp` filter
   - Params: `amount` (f64, 0.5-2.0), `radius` (f64, 1.0-5.0)
   - Clamp: amount [0.5, 2.0], radius [1.0, 5.0]
   - Format: `unsharp=luma_msize_x=3:luma_msize_y=3:luma_amount={amount}`
   - Returns: `vec!["-vf".to_string(), filter]`

**Geometric Fine-Tuning (3 functions):**

8. `build_micro_rotate_filter` — FFmpeg `rotate` filter (RESEARCH Code Examples, lines 652-658)
   - Params: `angle` (f64, degrees, -1.0 to 1.0 per D-01)
   - Clamp: angle to [-1.0, 1.0]
   - Convert degrees to radians using formula: `angle * std::f64::consts::PI / 180.0`
   - Format: `rotate={radians}:ow=iw:oh=ih` (preserve original dimensions)
   - Returns: `vec!["-vf".to_string(), filter]`

9. `build_tiny_scale_filter` — FFmpeg `scale` filter
   - Params: `scaleFactor` (f64, 0.99-1.01 per D-01)
   - Clamp: scaleFactor to [0.99, 1.01]
   - Compute: `new_w = (iw * {factor}) rounded`, use expression `iw*{factor}` directly
   - Format: `scale=iw*{scaleFactor}:ih*{scaleFactor}:flags=lanczos`
   - Returns: `vec!["-vf".to_string(), filter]`

10. `build_flip_filter` — FFmpeg `hflip` or `vflip` filter
    - Params: `direction` (String, "horizontal" or "vertical")
    - No numeric clamping needed
    - Format: `hflip` for "horizontal", `vflip` for "vertical"
    - Return error for unknown direction
    - Returns: `vec!["-vf".to_string(), direction_filter]`

**Blend Overlay (3 functions):**

11. `build_solid_color_overlay_filter` — FFmpeg `colorize` filter (RESEARCH Pattern 1, lines 252-270)
    - Params: `hue` (f64, 0-360), `saturation` (f64, 0-1), `lightness` (f64, 0-1), `mix` (f64, opacity)
    - Clamp: mix to [0.01, 0.15] per D-01 (opacity <= 0.15)
    - Format: `colorize=hue={hue}:saturation={saturation}:lightness={lightness}:mix={mix}`
    - Returns: `vec!["-vf".to_string(), filter]`

12. `build_gradient_overlay_filter` — FFmpeg `geq` filter with alpha expressions
    - Params: `type` (String, "linear" or "radial"), `opacity` (f64)
    - Clamp: opacity to [0.01, 0.15]
    - For "linear": generate geq with horizontal alpha gradient
    - For "radial": generate geq with radial alpha from center
    - Format (linear): `geq=r='r(X,Y)':g='g(X,Y)':b='b(X,Y)':a='alpha(X,Y)*(1-{opacity})+128*{opacity}*X/W'`
    - Experimentation note: The `geq` alpha expression for gradients is noted in RESEARCH as needing visual tuning. Use a working expression pattern; document that gradient quality may need refinement.
    - Returns: `vec!["-vf".to_string(), filter]`

13. `build_watermark_blend_filter` — FFmpeg `overlay` filter approach via `geq`
    - Params: `pattern` (String, "grid" or "diagonal"), `opacity` (f64)
    - Clamp: opacity to [0.01, 0.15]
    - Uses `geq` to create subtle pattern-based pixel modification
    - Format (grid): Create checkerboard-like luminance modulation at low opacity
    - Fallback: If `geq` expressions become too complex, use `noise=c0s=8:c0f=t+u:alls=8` with low strength as watermark-equivalent
    - Returns: `vec!["-vf".to_string(), filter]`
