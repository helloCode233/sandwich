## Spacing Scale

Declared values (all multiples of 4 from the standard set: 4, 8, 16, 24, 32, 48, 64):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, compact inline padding (`gap-1`) |
| sm | 8px | Compact element spacing (`gap-2`) |
| md | 16px | Default element spacing, general-purpose padding (`gap-4`, `p-4`) |
| lg | 24px | Section padding, card gaps (`gap-6`, `p-6`) |
| xl | 32px | Layout gaps, panel-level spacing (`gap-8`) |
| 2xl | 48px | Major section breaks, icon+text empty state vertical spacing |
| 3xl | 64px | Page-level spacing |

Exceptions:
- 12px (`gap-3`, `py-3`, `px-3`): Extensively used in existing components (SeedList header padding, QueueList item internal spacing, BatchControls footer padding) for compact desktop UI density. This is the UnoCSS/Tailwind intermediate step between 8px and 16px, providing readable density without crampedness. Preserved for backward compatibility with existing component patterns.
- 2px (`border-2`): used for selected seed card border thickness. Visual-only, not a layout spacing value.

**Source:** Existing codebase patterns verified across SeedCard.vue, QueueList.vue, BatchControls.vue, MainLayout.vue. All use UnoCSS utility classes with Tailwind-compatible spacing scale (1 = 4px).

## Visual Hierarchy

### Focal Point

The primary visual anchor depends on application state:

| State | Focal Point | Rationale |
|-------|-------------|-----------|
| First launch (empty queue, no seeds) | **SeedList header with "Generate Seed" primary button** in the left panel | The Generate button is the entry point to the core workflow -- without a seed, no processing is possible. It uses accent color `#2080f0` as `type="primary"`, the only colored button on screen against the dark `#101014` dominant surface. All other controls are secondary or disabled. |
| Queue populated, ready to process | **BatchControls footer "Start Processing" primary button** in the right panel | Once seeds exist and videos are queued, the Start button becomes the primary action driver. It is anchored at the bottom of the right panel (sticky), always visible regardless of queue scroll position, and uses accent color `#2080f0` exclusively. |
| Batch running | **NProgress bar on the active queue item** | During processing, the animated progress bar on the currently-processing file row draws attention to ongoing work. Accent color `#2080f0` on the active bar segment. |

The accent color `#2080f0` is never shared simultaneously across competing focal points -- when the Start button is primary, the Generate button uses a secondary type, and vice versa.

## Interaction States

### Seed Card Interactions

| State | Visual | Trigger |
|-------|--------|---------|
| Default | NCard with border-transparent | Initial render |
| Selected | NCard border-2 + `#2080f0`, Zap icon visible | Click to toggle |
| Hover | Action buttons fade in (opacity transition 200ms), existing pattern | Mouse enter |
| Hover + Selected | Both states combined | Mouse enter on selected card |
| Disabled | N/A (seeds always interactive except during batch) | -- |

**Phase 6 addition:** Strength tier badge (NTag, small, type="info") displayed below operation tags. Export/import buttons added to hover action bar alongside existing rename/copy/delete (CONTEXT D-11).

### Queue Item Interactions

| State | Visual | Trigger |
|-------|--------|---------|
| Default | Row with metadata line, status tag, remove button | Initial render |
| Active processing | NProgress bar inline (existing pattern from Phase 4) | `batch-file-progress` event for this file |
| Complete (100%) | NProgress color `#18a058` (existing) | `percent === 100` |
| Hover | No special state (existing) | -- |
| Drag active | SortableJS ghost element, drag handle cursor-grab | Drag handle mousedown (PROD-01) |
| Disabled (processing) | Drag disabled, remove button disabled | `batchStore.isProcessing` |

**Phase 6 additions:**
- Drag handle icon (GripVertical from lucide-vue-next) at left edge of each row (RESEARCH Pattern 4, Open Question 4).
- Thumbnail image (120px wide, object-cover, rounded) shown between drag handle and filename (PROD-02).
- Thumbnail loading: No explicit loading state needed -- extraction is synchronous within import command. If extraction fails, no thumbnail shown (graceful degradation).

### Strength Tier Selector

| State | Visual | Trigger |
|-------|--------|---------|
| Default | NSelect single-select, value="standard" | Initial render |
| Changed | New value persisted to shared store/ref | User selects option |
| Disabled | Grayed out | During active batch processing (`batchStore.isProcessing`) |

Placement: In BatchControls, between title and seed selector (RESEARCH Open Question 5 recommendation: conceptually a "processing intensity" setting, closer to batch controls than seed management).

### Log Panel

| State | Visual | Trigger |
|-------|--------|---------|
| Active tab | NTabs, "History" tab selected | User clicks tab |
| Entries present | Scrollable list with entry cards | Log entries exist in store |
| Empty | Centered icon + heading + body text | `logStore.entries.length === 0` |
| Filtering | NInput v-model for search, computed filtered list | User types in search |
| No filter results | "No entries match your filters" message | Computed list is empty but total > 0 |

Log panel placement: Right panel, as an NTabs sibling to the queue list. Tabs: "Queue" (default) | "History". The queue tab shows ImportZone + QueueList; the History tab shows LogPanel. (CONTEXT D-16, D-18: inline panel, not modal or drawer).

### BatchControls Flow

User flow for Phase 6 batch processing:

1. Select strength tier (new) -- determines how seeds are generated and what parameter ranges apply
2. Select seeds (existing, multi-select from Phase 5)
3. Set concurrency (existing)
4. Choose output directory (existing)
5. Click Start (existing)

If no seeds selected or queue empty, Start remains disabled (existing pattern).
