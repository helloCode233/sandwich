### Change 2: Add probe_global_metadata function

Insert after the `extract_metadata` function (after line 80, before the `// --- Raw ffprobe JSON structures ---` comment on line 82):

```rust
/// Run ffprobe to extract all global metadata tags from a video file.
/// Returns HashMap<key, value> of all metadata fields in the format.
/// Used by MetadataSelectiveErase to know which fields exist before targeted erasure.
/// This is separate from extract_metadata() because we only need it when
/// a MetadataSelectiveErase operation is present in the seed.
pub fn probe_global_metadata(filepath: &str) -> Result<std::collections::HashMap<String, String>, String> {
    let ffprobe_bin = ffprobe_path();
    let output = std::process::Command::new(&ffprobe_bin)
        .args(["-v", "quiet", "-print_format", "json", "-show_format", filepath])
        .output()
        .map_err(|e| format!("ffprobe failed: {}", e))?;

    if !output.status.success() {
        return Err("Cannot probe file metadata".to_string());
    }

    let probe: RawProbeOutput = serde_json::from_slice(&output.stdout)
        .map_err(|e| format!("Failed to parse ffprobe JSON: {}", e))?;

    Ok(probe.format.tags.unwrap_or_default())
}
```

Note: This function reuses the existing `RawProbeOutput` and `RawFormat` structs (already defined in this file). The `RawFormat.tags` field is added above.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05-01 | Tampering | executor FFmpeg args | mitigate | All FFmpeg arguments are passed via ffmpeg-sidecar's `.args(&Vec<String>)` which maps to `std::process::Command::args()` — no shell string construction. Each argument is a separate string in the vector, preventing injection. |
| T-07-05-02 | Tampering | probe_global_metadata filepath | mitigate | The filepath comes from `VideoEntry.filepath` which is set during video import (validated by ffprobe in extract_metadata). Not user-controlled at runtime. |
| T-07-05-03 | Denial of Service | ffprobe for MetadataContext | accept | probe_global_metadata runs a single ffprobe call once per file processing. For files without MetadataSelectiveErase, it's skipped entirely (lazy). The overhead is negligible compared to FFmpeg encoding. |
| T-07-05-04 | Tampering | -vsync vfr flag | accept | The -vsync vfr flag is injected unconditionally when FrameDrop is present. GPU encoder interaction with vfr is an accepted risk (RESEARCH Open Question 2). The flag is safe — it tells ffmpeg to use variable frame rate, which is the correct behavior when frames are dropped. |
