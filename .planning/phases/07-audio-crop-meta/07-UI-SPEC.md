## Visual Hierarchy

### Focal Point

The primary visual anchor depends on application state:

| State | Focal Point | Rationale |
|-------|-------------|-----------|
| First launch (empty queue, no seeds) | **SeedList header with "Generate Seed" primary button** in the left panel | The Generate button is the entry point to the core workflow -- without a seed, no processing is possible. It uses accent color `#2080f0` as `type="primary"`, the only colored button on screen against the dark `#101014` dominant surface. |
| Queue populated, ready to process | **BatchControls footer "Start Processing" primary button** in the right panel | Once seeds exist and videos are queued, the Start button becomes the primary action driver. It is anchored at the bottom of the right panel (sticky), always visible regardless of queue scroll position. |
| Batch running | **NProgress bar on the active queue item** | During processing, the animated progress bar on the currently-processing file row draws attention to ongoing work. |

The accent color `#2080f0` is never shared simultaneously across competing focal points.

## OperationType Expansion

Phase 7 adds the following OperationType variants. These display in SeedCard as NTag badges via the existing `visibleOps()` loop (max 3 visible, "+N more" overflow).

### Audio Operations (CONTEXT D-01, D-02, D-03)

Replaces the existing `audioTweak` variant with 5 focused types:

| OperationType (camelCase) | en Display | zh-CN Display | i18n Key |
|---------------------------|------------|---------------|----------|
| `audioResample` | "Audio Resample" | "音频重采样" | `operationTypes.audioResample` |
| `audioVolume` | "Audio Volume" | "音频音量" | `operationTypes.audioVolume` |
| `audioPitch` | "Pitch Shift" | "音调偏移" | `operationTypes.audioPitch` |
| `audioEQ` | "Equalizer" | "均衡器" | `operationTypes.audioEQ` |
| `audioChannel` | "Channel Op" | "声道操作" | `operationTypes.audioChannel` |

Safety constraints (CONTEXT D-02): Pitch +/-2 semitones, speed 0.97-1.03x, volume +/-3dB, resample 22050-48000 Hz. Pure FFmpeg built-in filters only.

### Crop (CONTEXT D-04, D-05, D-06, D-07, D-08)

| OperationType (camelCase) | en Display | zh-CN Display | i18n Key |
|---------------------------|------------|---------------|----------|
| `crop` | "Smart Crop" | "智能裁切" | `operationTypes.crop` |

Crop is a **default operation** (CONTEXT D-04): every seed guarantees one crop operation, plus crop can appear in the random pool for a potential second crop. Crop does not count toward the operation step count. It follows three-tier strength: Conservative 0.5-1.5%, Standard 1-2.5%, Aggressive 2-3.5% (CONTEXT D-07). Each edge (top, bottom, left, right) gets an independent random percentage within the tier range. Cropped output is scaled back to original resolution via crop+scale filter chain (CONTEXT D-06).

### Metadata Operations (CONTEXT D-09, D-10, D-11, D-12, D-13)

Existing `metadataErase` is retained. Two new types added:

| OperationType (camelCase) | en Display | zh-CN Display | i18n Key |
|---------------------------|------------|---------------|----------|
| `metadataWrite` | "Metadata Write" | "元数据写入" | `operationTypes.metadataWrite` |
| `metadataSelectiveErase` | "Selective Erase" | "选择性擦除" | `operationTypes.metadataSelectiveErase` |
| `metadataErase` | "Metadata Erase" | "元数据擦除" | `operationTypes.metadataErase` (existing) |

Metadata operations do **not** follow strength tiers -- all tiers behave identically (CONTEXT D-13). Fake metadata fields: creation_time (+/-30 days random offset), title, author, comment, copyright, encoder (from predefined word lists) (CONTEXT D-10, D-11). Selective erase categories: time (creation_time, modify_date), device (camera, make, model), description (title, comment, author, copyright). Randomly selects 1-3 categories to erase (CONTEXT D-12).

### Duration Modification (CONTEXT D-14, D-15, D-16)

| OperationType (camelCase) | en Display | zh-CN Display | i18n Key |
|---------------------------|------------|---------------|----------|
| `videoSpeed` | "Video Speed" | "视频变速" | `operationTypes.videoSpeed` |
| `trimEdges` | "Trim Edges" | "时长裁剪" | `operationTypes.trimEdges` |

VideoSpeed range: 0.95-1.05x (CONTEXT D-15). Uses setpts for video + atempo for audio synchronization. TrimEdges: randomly crops head, tail, or both, 1-30 frames per edge (CONTEXT D-16).

### FrameDrop Upgrade (CONTEXT D-17, D-18, D-19)

| OperationType (camelCase) | en Display | zh-CN Display | i18n Key |
|---------------------------|------------|---------------|----------|
| `frameDrop` | "Frame Drop" | "抽帧" | `operationTypes.frameDrop` (existing, behavior changed) |

FrameDrop is upgraded to a **default operation** alongside crop (CONTEXT D-19): every seed guarantees one FrameDrop, plus it can appear in the random pool. Reverts Phase 6's setpts micro-timing jitter approach back to true frame dropping via framestep filter (CONTEXT D-17). Drop interval: every 30-50 frames drop 1 frame. Follows three-tier strength: Conservative 40-50 frame interval, Standard 30-45 frame interval, Aggressive 25-35 frame interval (CONTEXT D-19).

### Operation Count Impact

With crop and FrameDrop as default operations (not counting toward step count, per CONTEXT D-04, D-19), the minimum visible operations per seed increases by 2. The `visibleOps()` function in SeedCard displays up to 3 ops before showing "+N more" -- with defaults, most seeds will show at least the 2 default ops plus 1-3 random ops, keeping most seeds within the 3-tag display window. Seeds at higher step counts will more frequently trigger "+N more".

## Interaction Contract (Unchanged)

All interaction patterns carry forward from Phase 06. Phase 7 introduces no new interactions:

### Seed Card Interactions

| State | Visual | Trigger |
|-------|--------|---------|
| Default | NCard with border-transparent | Initial render |
| Selected | NCard border-2 + `#2080f0`, Zap icon visible | Click to toggle |
| Hover | Action buttons fade in (opacity transition 200ms) | Mouse enter |
| Hover + Selected | Both states combined | Mouse enter on selected card |
| Disabled | N/A (seeds always interactive except during batch) | -- |

**Phase 7 note:** Seed cards now show more operation tags (minimum 2 from default ops: crop + frameDrop). The `overflowCount()` function handles >3 ops with "+N more" tag. No behavior change needed.

### Feedback Pyramid (Unchanged)

| Action Type | Feedback Mechanism | Examples |
|-------------|-------------------|----------|
| **Destructive** (irreversible) | NModal/NPopconfirm confirmation dialog | Delete seed, Clear queue, Cancel batch |
| **Mutating** (non-destructive) | useMessage() success toast | Generate seed, Copy seed, Rename seed, Import video |
| **Error** | useMessage() error toast + descriptive text | IPC failures, validation errors, import failures |
| **Progress** | Inline NProgress + BatchBanner | Batch processing |

### IPC Call Pattern (Unchanged)

All Tauri `invoke()` calls live in composables (`useSeed`, `useQueue`, `useBatch`), never in components directly.

### Event Listening (Unchanged)

| Event | Composable | Triggers |
|-------|-----------|----------|
| `seeds-updated` | useSeed | Re-fetch seed list via `list_seeds` |
| `queue-updated` | useQueue | Re-fetch queue via `get_queue` |
| `video-imported` | useQueue | Optimistic prepend to queue store |
| `batch-progress` | useBatch | Update batch store progress |
| `batch-file-error` | useBatch | useMessage() error toast per file |
| `batch-complete` | useBatch | useMessage() success + reset batch state |
| `batch-cancelled` | useBatch | useMessage() info + reset batch state |
| `low-disk-space` | useQueue | useNotification() warning |
